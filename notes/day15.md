## Why CRUD is core of backend ?
* its a core element everything required creation , reading, updation , deletion in any application you built so its a core of backend

## Why separate functions for each operation 
* code becomes clean
* easy to debug large codebase
* future updates become easier

## Why validation is critical 
* without it you code might fail without getting caught 
* its important for the security purpose too 

## Problem with using slice as DB 
* its a temporary memory 
* stored in RAM so its not persistent
* its fast but not reliable
